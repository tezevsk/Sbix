# Contributing to Sbix

Thank you so much! I'm glad that you're looking towards contributing to the project. 

To make it easier to make decisions and integrate your amazing code, please read the instructions below.

## How can I contribute?

### Reporting Bugs
* Check the **Issues** tab to see if the same problem has already been reported. 
* If not, feel free to open a new issue. It helps me and other contributors find the problem and fix it faster.

### Suggesting Enhancements
You can suggest things to add or enhance. Before doing so, please check that the feature:
1. Is not implemented yet.
2. Has not been suggested by someone else before.
3. Fits the general purpose of the project.

*Note: Suggestions that do not fit the project's purpose may be declined or blacklisted.*

### Pull Requests
Possibly the best thing you can do!
1. **Fork** the repository and clone it locally.
2. Create a new **branch** from `main` using a descriptive name: `category/what-it-is-made-for` (e.g., `feat/login-system` or `fix/button-click`).
3. Make your changes.
4. **Push** your branch to your fork.
5. Open a **Pull Request** against our `main` branch.

## If you need help
Feel free to open a thread in the **Discussions** tab.

# Thank You
And remember, no one is stopping you from doing all these things at once!
